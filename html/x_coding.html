Coding the XAML file

Most instruments will use a combination of shapes, buttons, or text-blocks in order to convey their purpose. The classes and methods utilized are determined by the ultimate goal of the instrument. The shape class has a variety of geometric sub classes which can be used to create the design of the instrument. The button class can be used to trigger events. The text-block class is used to display text. There are a plethora of other classes that can be utilized and are not covered in this document. Often times the web will have the answer or a starting point. The main point of the .XAML file is to develop the interface of the instrument.

The code below shows the XAML code for the Ribbon Compass Instrument. The goal of the Ribbon Compass was to display the heading of the ROV. It therefore only needs to show the output of a parameter from the ROV. The Ribbon Compass does not need to send information back to the ROV, making it simpler than Compass and Attitude Indicator Instrument. The shape classes include the "rectangle class, the polygon class and the line class. The line class is not covered in this document because it is so similar to the other shape classes. The ellipse class is an example of an another shape class not used in the Ribbon Compass example. The text-block class is the other class utilized in the the Ribbon Compass Instrument example.
