Fill Property and ImageBrush Class

Fill is a property that gets or sets the Brush, or how the shape’s (used with the rectangle class in the example) interior is painted. A color is therefore a valid value for the fill property. ImageBrush is also a valid value for the fill property. ImageBrush takes an image and paints it on the specific shape. For more information on the Fill property click here. For more information on the ImageBrush class click here.

ImageSource

ImageSource specifies where the image is coming from for the ImageBrush class. The path of the image can be from any folder on the computer, with the correct path. Without a "C:" or likewise prefix, the folder path assumes that it starts at the Project’s name’s folder. So in the example’s case the "InstrumentHostSample" folder. For information on the ImageSource Property click here.

How to Make an Image a Resource

The reason to make your image a resource is so it can be referenced from any computer and not have to be hard coded. To add a file as a resource:

  1. Expand your Project (Your project's name) in "Solution Explorer"
  2. Right click on "Resource"
  3. Select "Add"
  4. Select "Existing Item… "
  5. Navigate and select your image (you may have to change the type of files which can be selected)
  6. Click "Add"
  7. Expand the "Resources" folder in the "Solution Explorer" within your expanded project
  8. Select "Your image" by left clicking on it
  9. In the "Properties" Window (View> Properties if you cannot see it) select "Resource" from the dropdown menu for "Build Action"

The Image

Visual Studio is rather robust and can take a variety of image types, including but not limited to jpeg and png. While not usually a problem, the programmer may want to be cognisant of the size of there image.

The Ribbon Compass example utilizes a png file as a resource. The resource is a part of the project and can be referenced from any computer (not having to be hard coded). In the example, an image was made in paint and used as the image brushed onto the rectangle. The paint png file was created from 24 equal parts each representing 45 degrees of a compass (45*24= 1080). The value of 1080 was chosen to allow for flexibility for viewing. The updating section helps explain the manipulation of the Yaw property. The Yaw property is from 0 to 359 and the C# code will move the rectangle 1 position to the left for every increment in degree. So 90 degrees moves the rectangle's position 90 to the left and making the compass needle appear over East (mimicking a right turn from North). At 359 it just skips back to the 0 position (seamlessly) so that the same 360 positions are used infinitely. The image was made to be 1080, or 3 full compasses, so that if you wanted, you could easily set the boundaries of the ribbon compass to view 360 on either side of the needle.
