Starting the XAML coding

In the XAML window is where simple coding is done and the Designer window displays what it will look like. For more overview of what the Designer provides click here. For more overview of what the XMAL provides click here. This section will continue with renaming your XAML file and other first steps.

To rename your file:

  1. In "Solution Explorer" expand "Your Instrument's name" project
  2. Right click on "MainWindow.xaml"
  3. Select Rename
  4. Type in the highlighted area "Your Instrument's name"
For the rest of the set up:
  1. In "Solution Explorer" expand "Your Instrument's name"
  2. Double click on ""Your Instrument's name".xaml"
  3. The XAML and Design Window are now open. Edit the XAML window to read (Substitute "Your Instrument's name" for "RibbonCompass" on the first line):
Between the "Canvas" class is where one begins to write their code. The "Canvas" class defines an area within which you can explicitly position child objects, using coordinates that are relative to the Canvas area. Important to note: Height="10" Width="100" are the dimensions visible once the instrument is running. The dimensions shown in Cockpit can be altered by the C# code, but the Height and Width located here are what show in the Designer window. Click here for more on the "Canvas" class.
